# @sebse-tools/n8n-nodes-ics-parser

![version](https://img.shields.io/badge/version-1.0.1-blue) ![repo](https://img.shields.io/badge/monorepo-source-blue?logo=github&link=https%3A%2F%2Fgithub.com%2FCaesar2011%2Fmcp-monorepo%2Ftree%2Fmain%2Fpackages%2Fn8n-nodes-ics-parser)

n8n node to parse ICS calendar files and query events

This node integrates [`@sebse-tools/ics-parser`](https://www.npmjs.com/package/@sebse-tools/ics-parser) into n8n workflows, enabling parsing and querying of iCalendar (`.ics`) data directly inside your automations.

## Operations

### Prepare ICS

Parses a raw ICS calendar into a structured `PreparedIcs` JSON object. Use this as the first step before querying events.

| Parameter             | Type    | Description                                                                         |
| --------------------- | ------- | ----------------------------------------------------------------------------------- |
| Input Type            | Options | `Binary File` or `ICS String`                                                       |
| Input Data Field Name | String  | Binary only. Name of the binary field on the input item containing the `.ics` file. |
| ICS String            | String  | String only. Raw ICS calendar content.                                              |

**Output.** One item per input item. The `json` payload is the full `PreparedIcs` object, which can be passed directly to the `Get Events Between Dates` operation.

### Get Events Between Dates

Queries a prepared ICS object for all events within a given date range, including all recurring event instances.

| Parameter    | Type     | Description                                                   |
| ------------ | -------- | ------------------------------------------------------------- |
| Prepared ICS | JSON     | The `PreparedIcs` object from a preceding `Prepare ICS` node. |
| Start Date   | DateTime | Start of the date range, inclusive.                           |
| End Date     | DateTime | End of the date range, inclusive.                             |

**Output.** One item per event occurrence found in the range.

## Usage

### Typical workflow: Fetch, Prepare, Query

```
HTTP Request  →  ICS Parser (Prepare ICS)  →  ICS Parser (Get Events Between Dates)
```

1. **HTTP Request node.** `GET` your calendar URL, e.g. a CalDAV public share or a `.ics` file URL. Set **Response Format** to `File` to receive binary output, or `Text` to pass the raw string.

2. **ICS Parser, Prepare ICS.**

- Binary file input: set **Input Type** to `Binary File` and **Input Data Field Name** to `data`.
- Text input: set **Input Type** to `ICS String` and map the response body to **ICS String** via an expression, e.g. `{{ $json.body }}`.

3. **ICS Parser, Get Events Between Dates.**

- Set **Prepared ICS** to `{{ $json }}` to map the full output of the previous node.
- Set **Start Date** and **End Date** using expressions or fixed values, e.g. `{{ $now }}` and `{{ $now.plus(7, 'days') }}`.

### Caching the prepared object

Because `Prepare ICS` is computationally heavier than `Get Events Between Dates`, you can store the output of `Prepare ICS`, e.g. in a database or n8n static data, and reuse it across multiple `Get Events Between Dates` calls without re-parsing the source file.

### Example. Weekly agenda digest

```
Schedule Trigger (every Monday)
  → HTTP Request (fetch calendar .ics)
  → ICS Parser, Prepare ICS (Binary File, field "data")
  → ICS Parser, Get Events Between Dates
      Prepared ICS:  {{ $json }}
      Start Date:    {{ $now.startOf('week') }}
      End Date:      {{ $now.endOf('week') }}
  → Send Email, Slack, etc.
```
